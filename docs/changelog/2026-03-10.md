# 2026-03-10

## 1) Dev and Docker workflow standardization
- local 개발 경로를 `pnpm dev` + `pnpm infra:down` 기준으로 정리하고, `infra:down`은 `db`/`redis`만 중지/제거하도록 수정
- Dockerfile 검증 경로를 `pnpm docker`, `pnpm docker:logs`, `pnpm docker:down`으로 정리
- `package.json` 스크립트는 `setup -> dev -> infra -> docker -> quality -> nx` 순서로 재배치

## 2) Runtime and environment reproducibility
- Node `24.11.0`, Python `3.11.15`, Postgres `16.13-alpine`, Redis `7.4.7-alpine`로 patch 버전까지 고정
- app 로컬 env는 `apps/*/.env.example`만 추적하고, 실제 `.env`는 실행 시 자동 생성하도록 전환
- Compose 데모용 env는 루트 `compose.env`로 정리하고 섹션 주석을 추가

## 3) Docker build and cache improvements
- frontend / backend Dockerfile 추가 및 BuildKit 기반 빌드 경로 정리
- `pnpm docker`는 Nx stream 출력 + `docker build --progress=plain`으로 단계 로그를 추적 가능하게 조정
- frontend Dockerfile은 의존성 설치 레이어 캐시가 재사용되도록 `package.json` 기반으로 분리
- Docker와 로컬이 `node_modules`, `.venv`, Vite cache를 공유하지 않도록 경로를 분리해 권한 충돌을 방지

## 4) Documentation and project guidance refresh
- 루트 `CHANGELOG.md`를 제거하고 `docs/changelog/YYYY-MM-DD.md` 구조로 전환
- README를 외부 학습자가 바로 실행/이해할 수 있는 형태로 전면 재구성
- AGENTS는 실제 운영 규칙, 실행 정책, env 정책 중심으로 다시 정리
