## 2. Verification Became A Real Access-Control Flow

### What Changed

- Invite-code approval was added so accepted users can be auto-approved immediately.
- New or rejected applicants now need a matching invite code, while already approved users can still edit without being downgraded.
- The full signed-up-user directory under recommendations is now hidden until approval.
- `/verification` also gained a self-serve account deletion flow protected by email re-entry.

### Why It Mattered

- The demo now has a clearer boundary between public curiosity and approved-member access.
- Verification is no longer just a form; it now controls what users can actually see and do.
- Account deletion makes the flow feel more complete and safer for real testers.
