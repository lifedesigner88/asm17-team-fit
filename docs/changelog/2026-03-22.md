## 1. pnpm Bootstrap тАФ Auto-Install via npm

### What Changed
- `scripts/setup-dev.mjs`: added pnpm presence check at the top of the script
  - If `pnpm --version` fails, reads the `packageManager` field from `package.json` and runs `npm install -g pnpm@<version>` automatically
- `README.md` Quick Start changed from `pnpm setup` тЖТ `node scripts/setup-dev.mjs` as the first-run command

### Why It Mattered
- Fresh Ubuntu 24 has no pnpm. `pnpm setup` requires pnpm to already exist тАФ a circular dependency.
- `node scripts/setup-dev.mjs` only requires Node (bundled with nvm), so it can bootstrap pnpm itself.

## 2. `pnpm install` Moved Inside setup-dev.mjs

### What Changed
- `pnpm install` call added directly inside `setup-dev.mjs` (after `ensure-dev-env`, before `uv sync`)
- Removed the `&& pnpm install` suffix that was appended to the `package.json` `setup` script

### Why It Mattered
- pnpm skips or warns when `node_modules` is missing and the shell `&&` chain is not reliable in that state.
- Running `pnpm install` via `spawnSync` inside the Node script is unconditional and always executes.
