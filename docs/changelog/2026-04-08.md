## Startup Command Fix

### What Changed

- Added a new `pnpm bootstrap` script that forwards to `pnpm run setup`.
- Updated the README local run section to use `pnpm bootstrap` instead of bare `pnpm setup`.

### Why It Mattered

- On pnpm `10.31.0`, `pnpm setup` executes pnpm's own CLI command instead of the repo script.
- That left `node_modules` uninstalled, which caused the later `nx: not found` failure during `pnpm dev`.

### Files To Read

- `package.json`
- `README.md`

## Friendlier Dev Prereq Failure

### What Changed

- Added `scripts/ensure-local-tooling.mjs`.
- Wired `pnpm dev` and `pnpm docker` to check for the local `nx` binary before launching Nx.

### Why It Mattered

- Missing dependencies now fail with a direct instruction to run `pnpm bootstrap` or `pnpm run setup`.
- This replaces the low-signal shell error with a repo-specific recovery path.

### Files To Read
